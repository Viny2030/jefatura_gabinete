Documentación Técnica

Monitor de Transparencia del Poder Ejecutivo Nacional — arquitectura, endpoints y referencia de datos

⚠️ Aviso legal: Este sistema utiliza exclusivamente datos públicos provenientes de fuentes oficiales del Estado Nacional Argentino (comprar.gob.ar, BORA, MapaDelEstado). No contiene datos sensibles de personas físicas. Los cruces y alertas son indicadores estadísticos que no implican presunción de ilicitud. El uso de esta herramienta no reemplaza la investigación periodística ni el proceso judicial formal.

⚙️ Arquitectura del sistema

Pipeline automatizado que corre diariamente vía GitHub Actions, persiste datos en PostgreSQL (Railway) y sirve el frontend via FastAPI.

ComponenteTecnologíaDescripción
scraper_comprar.pyPython / BeautifulSoupExtrae contratos de comprar.gob.ar (ASP.NET UpdatePanel)
scraper_nomina.pyPython / requestsNómina APN desde MapaDelEstado — 6.000+ funcionarios
generar_cruces_pen.pyPython / pandasCruza CUILs de nómina contra CUITs de proveedores
pipeline.pyPythonOrquesta scrapers → matrices XAI → inteligencia.json
api_server.pyFastAPISirve endpoints REST y HTML estático
GitHub ActionsYAML / cronEjecución diaria automática (lun–vie 12:00 UTC)
RailwayPostgreSQL / DockerBase de datos y hosting del servicio web

🚨 Niveles de alerta
ALTO

Match exacto CUIL/CUIT entre funcionario y proveedor adjudicado en el mismo organismo.

MEDIO

Coincidencia de apellido en razón social, o proveedor multi-organismo con nómina activa.

BAJO

Indicadores estadísticos: montos atípicos, patrones de fragmentación, contratos repetidos.
