03
Manual de Usuario
Guía completa para navegar y aprovechar el Monitor XAI
📊

Dashboard Principal

Vista global con KPIs del Poder Ejecutivo Nacional: contratos totales, montos adjudicados, alertas activas y proveedores únicos monitoreados.

🚀

Análisis en Vivo

Procesamiento en tiempo real de nuevas adjudicaciones publicadas en comprar.gob.ar. Detecta anomalías automáticamente al actualizar los datos.

🔔

Alertas

Panel de alertas clasificadas por nivel (Alta / Media / Baja). Incluye sobreprecio, concentración de proveedores y posibles conflictos de interés.

🏛

Organismos (JGM · SGP · PNA)

Fichas individuales por organismo con sus contratos, nómina de autoridades, cruces detectados y estadísticas de gasto desagregadas.

🕸

Grafo de Relaciones

Visualización interactiva de vínculos entre funcionarios y proveedores. Permite detectar redes de relaciones potencialmente conflictivas.

⚡

API REST

Acceso programático a todos los datos mediante endpoints documentados en /api/docs. Ideal para periodistas y analistas de datos.

Cómo usar el sistema

1

Seleccionar un organismo

En el Dashboard hacé clic en la card del organismo que querés analizar: JGM, SGP o Presidencia. Cada uno tiene datos independientes.

2

Explorar contratos

Dentro de la ficha del organismo encontrarás la tabla de contratos con filtros por tipo, estado, proveedor y rango de monto. Podés exportar a CSV con un clic.

3

Revisar alertas generadas

El algoritmo XAI genera alertas automáticas. Una alerta ALTA indica: sobreprecio >80% sobre la mediana del rubro, o funcionario con CUIL idéntico a CUIT de proveedor adjudicado.

4

Usar el Grafo de Relaciones

Navegá a Grafo de Nodos para visualizar las conexiones. Los nodos azules son funcionarios, los rojos son proveedores. Las aristas muestran el tipo de vínculo detectado.

5

Acceder a la API para análisis avanzados

Consultá /api/docs para la documentación Swagger interactiva. Todos los endpoints son de libre acceso (GET) sin autenticación requerida.

Referencia rápida de la API

Método Endpoint Descripción Tags
GET /api/v1/kpis KPIs globales del sistema Dashboard
GET /api/v1/alertas Listado de alertas activas (filtrable por nivel y tipo) Alertas
GET /api/v1/contratos Contratos con filtros por organismo y proveedor Contratos
GET /api/v1/contratos/comprar Procesos scrapeados de comprar.gob.ar Contratos
GET /api/v1/contratos/top-proveedores Ranking de proveedores por monto adjudicado Contratos
GET /api/v1/nomina Nómina de funcionarios con cargo y organismo Nómina
GET /api/v1/grafo/nodos Nodos y aristas del grafo de relaciones Grafo
GET /api/v1/health Estado del sistema y conexión a base de datos Sistema
💡 Tip: Para cruzar datos de forma programática, combiná /api/v1/nomina?organismo=JGM con /api/v1/contratos?organismo=JGM y buscá CUILs de funcionarios que aparezcan como CUIT de proveedor. El sistema ya hace esto automáticamente con la Matriz Societaria.

Algoritmos de detección (Motor XAI)

👨‍👩‍👧

Matriz de Parentesco

Cruza apellidos de la nómina contra tokens de razón social de proveedores del mismo organismo. Alerta ALTA cuando hay coincidencia de apellido + organismo.

🏢

Matriz Societaria

Detecta cuando el CUIL de un funcionario es idéntico al CUIT de un proveedor adjudicado. Indica que una persona física es simultáneamente empleado estatal y proveedor.

💰

Matriz de Flujo de Fondos

Identifica sobreprecios (>40% sobre mediana del rubro) y concentración de proveedores (>30% del gasto de un organismo en un solo proveedor).
